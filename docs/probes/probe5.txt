- Coupling conclusion:
  - Proven: `UnifiedStudio.tsx` is too coupled to safely add features without high regression risk.
  - Reason: upload, validation, model selection, session persistence, generation, result display, feedback, and restart/regenerate behavior are all coordinated through many independent `useState` variables in one component.
  - Strongest evidence:
    - line 800 session persistence dependency array mixes 6 upload/model/validation states.
    - line 962 paste handler crosses upload/model phases.
    - lines 994-1090 generation consumes and mutates upload/model state directly.
    - lines 1206-1222 one reset handler clears multiple phases.
    - line 1644 leaves `jewelryAssetId` uncleared while clearing related upload state.
