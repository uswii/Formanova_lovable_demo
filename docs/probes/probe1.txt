- Risk
  - Highest breakage risk is src/pages/UnifiedStudio.tsx:
    - It combines upload flow, guide UI, presets, generation orchestration, dialogs, polling, error states, and authenticated image access in one 2,233-line page.
  - CAD changes are risky because src/pages/TextToCAD.tsx and src/components/text-to-cad/CADCanvas.tsx both own large chunks of workflow state, polling, downloads, rendering, and auth-sensitive asset fetching.
  - Backend integration changes can easily drift because there are several fetch styles:
    - authenticatedFetch
    - raw fetch
    - manual token headers
    - hardcoded https://formanova.ai
    - relative /api, /auth, /billing
    - env-based VITE_PIPELINE_API_URL
  - Auth/session behavior is coupled across:
    - src/lib/auth-api.ts
    - src/lib/authenticated-fetch.ts
    - src/contexts/AuthContext.tsx
    - src/components/ProtectedRoute.tsx
    - route redirects in src/App.tsx
  - Admin and credits flows are fragile because they mix React Query, manual fetchers, admin secrets, bearer tokens, modals, and nested routes across:
    - src/pages/AdminFeedbackPage.tsx
    - src/pages/PromoAdminPage.tsx
    - src/lib/credits-api.ts
    - src/contexts/CreditsContext.tsx

- Probe 1 interpretation
  - There is no single way to do a backend API call.
  - The project has grown without a strong integration boundary.
  - AI-assisted feature additions likely copied nearest examples and introduced multiple styles.
  - The main technical symptom is missing enforced convention for:
    - where API clients live,
    - when to use React Query,
    - when to use authenticatedFetch,
    - how auth failures are handled,
    - how base URLs are resolved,
    - how polling workflows are modeled,
    - how errors are normalized.

- Core takeaway
  - The repo does not just have messy files.
  - It has features whose state, API behavior, auth behavior, UI behavior, and routing behavior are tangled together.
  - That coupling is why changes cause regressions across multiple things.
