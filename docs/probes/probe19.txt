Suggested env var set

- `VITE_API_BASE_URL`
  - Owns backend base URL when absolute URL is truly needed.
  - Prefer relative `/api/...` with `authenticatedFetch` for same-origin protected API calls.

- `VITE_PUBLIC_SITE_URL`
  - Owns canonical/legal/public site links.
  - Example values:
    - production: `https://formanova.ai`
    - staging: `https://staging.formanova.ai`

- `VITE_AZURE_BLOB_BASE_URL`
  - Owns generic Azure blob storage base.
  - Example:
    - `https://snapwear.blob.core.windows.net`

- `VITE_MODEL_ASSET_BASE_URL`
  - Owns model-library asset root.
  - Example:
    - `https://snapwear.blob.core.windows.net/agentic-artifacts/Models`
  - Better long-term option:
    - backend returns model asset URLs so frontend does not know Azure account/container paths.

- `VITE_WEIGHT_STL_ALLOWLIST_EMAILS`
- `VITE_CAD_UPLOAD_ALLOWLIST_EMAILS`

Core finding

- Staging and production are not fully separable by environment config today.
- The biggest blockers are not SEO/legal links.
- The real blockers are:
  - production backend URLs inside runtime API paths,
  - Azure blob account/container paths in frontend libs,
  - internal email allowlists hardcoded in frontend feature flags.
