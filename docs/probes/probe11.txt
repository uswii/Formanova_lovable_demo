- `UnifiedStudio.tsx` actual polling loop:
  - Location:
    - `src/pages/UnifiedStudio.tsx:1109-1168`
  - Start source:
    - workflow starts through:
      - `startPdpShot(...)`
      - or `startPhotoshoot(...)`
    - both return `workflow_id`.
  - Endpoint polled:
    - `/api/status/{workflowId}`
    - called through `getPhotoshootStatus(startResponse.workflow_id)`.
  - Interval:
    - 3 seconds.
  - Timeout:
    - `TIMEOUT = 720000`
    - 12 minutes.
  - Exit conditions:
    - success:
      - `resolveWorkflowState(status) === 'completed'`
      - then fetch result via `getPhotoshootResult(startResponse.workflow_id)`.
    - failure:
      - `resolveWorkflowState(status) === 'failed'`
      - throws `status.error || 'Photoshoot generation failed'`.
    - timeout:
      - loop exits after 12 minutes and throws.
    - result-level failure:
      - after status completed, result is scanned for activity errors:
        - any item with `action === 'error'` or `status === 'failed'`
      - then sets `generationError = 'workflow-failed'`.
  - Progress behavior:
    - status progress fields can move progress from 35% to 95%.
    - a separate 300ms ticker moves apparent progress toward 90% even before real progress data arrives.
  - Coupling note:
    - polling, progress UI, result fetching, result parsing, credit refresh, analytics, and phase transition all live in the page.

- `src/lib/photoshoot-api.ts` polling/retry behavior:
  - Does it poll workflow status?
    - no.
  - Does it own a full poll-until-complete loop?
    - no.
  - Status helper:
    - `src/lib/photoshoot-api.ts:105`
      - `getPhotoshootStatus(workflowId)`
      - single GET to `/api/status/{workflowId}`.
    - `src/lib/photoshoot-api.ts:113`
      - returns `{ state: 'running' }` on 404.
  - Result helper:
    - `src/lib/photoshoot-api.ts:127`
      - `getPhotoshootResult(workflowId, maxRetries = 5, retryDelayMs = 1000)`
    - endpoint:
      - `/api/result/{workflowId}`
    - retry loop:
      - retries only result fetches after status completed.
    - interval:
      - default 1 second.
    - max attempts:
      - attempts `0` through `maxRetries`, so default is up to 6 total attempts.
    - retry condition:
      - HTTP 404 means result not written yet.
    - exit conditions:
      - success:
        - first OK response returns JSON.
      - failure:
        - non-OK non-404 throws.
      - retry exhausted:
        - throws last 404 error or generic exhausted error.
  - Conclusion:
    - `photoshoot-api.ts` has result-write retry, but not workflow polling.
    - `UnifiedStudio.tsx` polls inline.

- Shared polling hook design requirements implied by the audit:
  - Accept:
    - `workflowId`
    - status endpoint builder
    - result endpoint builder
    - auth fetcher strategy:
      - `authenticatedFetch`
      - or direct fetch with headers
    - poll interval
    - timeout
    - allowed 404 count policy
    - allowed transient error count
    - terminal state parser
    - terminal node parser
    - result readiness parser
    - abort signal/cancel support
    - `onProgress` callback
    - `onNodeChange` callback
  - Return:
    - status data
    - parsed state
    - result data
    - progress info
    - error
    - cancel function
  - Must support at least three current workflow styles:
    - photo/on-model:
      - poll `/api/status`
      - then retry `/api/result` on 404
    - CAD generation/edit:
      - poll `/api/status`
      - terminal node support
      - then retry `/api/result` on 404
    - weight/STL:
      - poll `/api/result` directly until status is not `running`
  - Should not own product-specific result parsing:
    - photo image extraction should stay in a photo parser
    - CAD GLB/STL/weight extraction should stay in CAD parsers
    - hook should only coordinate lifecycle and cancellation.

- Core finding:
  - There is no single current polling abstraction for generation workflows.
  - The deleted `workflow-api.ts` and `temporal-api.ts` are not the current problem because they no longer exist.
  - The live problem is duplicated inline polling across `UnifiedStudio.tsx`, `TextToCAD.tsx`, and `use-image-validation.ts`, plus partial retry logic inside `photoshoot-api.ts`.
  - A shared polling hook for the new zone should replace inline `while + setTimeout` loops and should separate lifecycle polling from product-specific result parsing.
