- Pricing conclusion:
  - `TOOL_COSTS` is not a true single source of truth.
  - It is a fallback table for CAD costs, plus some stale keys.
  - Backend estimate appears to be the real authority for live preflight/display where wired.
  - Live photo workflow key is missing from the table.
  - Several pricing concepts are mixed:
    - backend workflow names
    - legacy workflow names
    - UI fallback costs
    - history classifier names
    - model-specific cost variants

- Feature flag conclusion:
  - `feature-flags.ts` is a mix of:
    - real email gates
    - constant rollout flags that are no longer real toggles
    - unused/comment-only leftovers
  - Constant flags used by live files create dead branches and misleading architecture.
  - For a real single source of truth:
    - permanent `true` flags should be removed and their enabled path made normal code
    - permanent `false` flags should either be deleted with the hidden UI or converted to a real config/remote flag
    - email-gated internal flags should be grouped separately from product rollout flags
    - pricing flags and pricing keys should not live as unrelated ad hoc constants.

- Core finding:
  - Pricing and flags are not yet single-source-of-truth systems.
  - Pricing has stale keys, missing live workflow keys, dynamic model fallbacks, and backend-estimate overrides.
  - Feature flags have at least 10 constant true/false toggles still imported by live files.
  - This explains why adding or changing a paid feature requires touching multiple places and still risks inconsistent UI, cost display, and enforcement.
